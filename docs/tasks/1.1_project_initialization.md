# 1.1. Project Initialization

**Goal**: Establish the Go module and directory structure following the "Standard Go Project Layout" and preparing for Cobra/Bubble Tea.

## 1. References & Context
*   **PRD**: [Directory Structure](../PRD.md#41-directory-structure)
*   **Standards**:
    *   [Standard Go Project Layout](https://github.com/golang-standards/project-layout)
    *   [Cobra Generator Docs](https://github.com/spf13/cobra-cli/blob/main/README.md)
*   **Libraries**:
    *   [Cobra](https://github.com/spf13/cobra): For the CLI entrypoint.
    *   [Bubble Tea](https://github.com/charmbracelet/bubbletea): For the TUI (will live in `internal/tui/`).

## 2. Detailed Steps

### A. Module Initialization
Initialize the Go module.
```bash
go mod init github.com/rtfa/kevin
```

### B. Directory Structure
Create the following directory tree. This structure separates the "Application" (`cmd/`) from the "Library" (`internal/`) and "Domain" (`core/`).

```text
kevin/
├── cmd/
│   └── kevin/
│       └── main.go       # Entrypoint (Cobra root)
├── internal/
│   ├── core/             # Domain Models (Task, Config) - Pure Go, no dependencies
│   ├── store/            # Data Layer (fsnotify, frontmatter)
│   ├── executor/         # Agent Runners (Docker/Local)
│   ├── tui/              # Bubble Tea Views & Models
│   └── cli/              # Cobra Command Implementations
└── .kevin/               # (Gitignored or Example) Local dev storage
```

### C. Dependencies
Pre-fetch the core dependencies we know we need.
```bash
go get -u github.com/spf13/cobra
go get -u github.com/charmbracelet/bubbletea
go get -u github.com/charmbracelet/lipgloss
go get -u github.com/fsnotify/fsnotify
go get -u github.com/adrg/frontmatter
```

## 3. Success Criteria
- [ ] `go.mod` exists and valid.
- [ ] Directory structure created.
- [ ] `cmd/kevin/main.go` exists (can be empty hello world for now).
- [ ] Project compiles (`go build ./cmd/kevin`).
